# Code of Conduct

Project NEXUS is built for communities. The project space should be practical, respectful, and safe for people with different backgrounds, skill levels, and lived experience.

## Expected Behaviour

- Be respectful, patient, and constructive.
- Assume good intent while still being clear about problems.
- Give feedback about the work, not the person.
- Credit ideas, code, research, testing, and translation help accurately.
- Be careful when discussing vulnerable users, safeguarding, accessibility, identity, and personal data.

## Unacceptable Behaviour

- Harassment, threats, hate speech, or personal attacks.
- Sexualized language or imagery in project spaces.
- Doxxing, sharing private information, or pressuring people to reveal personal details.
- Deliberately derailing issues, pull requests, reviews, or support threads.
- Publishing unpatched security vulnerabilities instead of following `SECURITY.md`.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; reject contributions; or restrict participation when behaviour harms the project or its contributors.

If you need to report conduct concerns, contact the maintainer through the public Project NEXUS contact channel. Security issues should use the private process in `SECURITY.md`.
